You are on your way to a better, faster eCommerce platform.

However, if you are seeing this page, it means you are using IP Address, not virtual host!

This is probably not the page you want.

This page is being served from /var/www/html/index.html, and is the default vhost (virtual host) that comes with most web servers.

What you probably need to do is to add a vhost record on your machine to get this to run as intended.

On OSX and Linux, this is located at /etc/hosts, on Windows it is located at C:\Windows\System32\drivers\etc\hosts.

Add following line with the IP address of load balancer to your hosts file to be able to see what we're working on. 

                        192.168.56.100 star.commerce.dev
                        192.168.56.102 magento2.api.provider

                        192.168.56.120 magento2distributed.dev

                        192.168.56.200 whatever.local       # will search index.php in whatever/ directory
                        192.168.56.200 whatever.public      # will search index.php in whatever/public/ directory
                        192.168.56.200 whatever.webroot     # will search index.php in whatever/webroot/ directory
                        192.168.56.200 whatever.public_html # will search index.php in whatever/public_html/ directory
